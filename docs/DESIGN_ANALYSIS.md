## 2. Sand Dune 프로토타입의 장점 (컨셉 목업 대비)

### 2.1 구조적 차이

| 관점 | 컨셉 목업 | Sand Dune 프로토 | 차이 |
|------|----------|-----------------|------|
| **화면 크기** | 380px 고정 | 80vh 풀스크린 | 실제 비율 확인 가능 |
| **뷰 수** | 1개 (문서관리만) | 5개 (문서, 데이터소스, RAG채팅, 인덱싱, 설정) | 전체 UX 플로우 검증 |
| **인터랙션** | 없음 | 탭 전환 JS | 네비게이션 경험 확인 |
| **아이콘** | 이모지/텍스트 | Lucide 벡터 아이콘 | 프로덕션 품질 |
| **타이포그래피** | 시스템 폰트만 | Google Fonts (Playfair + Noto Sans KR) | 브랜드 아이덴티티 |
| **애니메이션** | 없음 | fadeIn 트랜지션 | 체감 품질 향상 |

### 2.2 콘텐츠 완성도

| 뷰 | 컨셉 목업 | Sand Dune 프로토 |
|----|----------|-----------------|
| 문서 관리 | 업로드존 + 카드 2~3개 | 업로드존 + 파일목록 (아이콘별 구분, 상태뱃지) |
| 데이터 소스 | 없음 | 4개 커넥터 카드 (Slack, Crawler, Notion, PostgreSQL) |
| RAG 채팅 | 없음 | 채팅 UI + 출처 인용 블록 + 하이라이트 |
| 인덱싱 | 없음 | 프로그레스바 + 통계 그리드 + 벡터DB 상태 |
| 설정 | 없음 | 플레이스홀더 |

### 2.3 핵심 장점 요약

1. **멀티뷰 프리뷰** — 한 테마로 전체 앱 UX 시뮬레이션 가능
2. **RAG 출처 UI** — 채팅 답변에 출처 문서 인용 + 키워드 하이라이트 (mark 태그)
3. **인덱싱 모니터** — 프로그레스바, 통계 대시보드, 벡터DB 상태 카드
4. **데이터 소스 카드** — 연결 상태 뱃지가 있는 커넥터 카드 패턴
5. **Tailwind 기반 컬러 시스템** — `tailwind.config`에 커스텀 팔레트 정의

## 3. 실제 대시보드 vs Sand Dune 프로토타입

### 3.1 실제 대시보드 (dashboard/) 현황

- **스택**: React + Vite + TypeScript + Tailwind CSS v4
- **폰트**: Pretendard Variable
- **테마**: Light/Dark 토글 지원
- **컬러**: `indigo-600` 액센트, `slate`/`zinc` 그레이
- **라우팅**: React Router (documents, collections, search, graph, settings)
- **API**: 실제 백엔드 연동 (`api.ts`)
- **i18n**: 한/영 전환 지원

### 3.2 Sand Dune 프로토가 가진 추가 장점 (실제 대시보드에 없는 것)

| 기능 | 현재 대시보드 | Sand Dune 프로토 |
|------|-------------|-----------------|
| RAG 채팅 출처 인용 UI | 검색결과 리스트만 | 채팅 + 인라인 출처 블록 + 하이라이트 |
| 인덱싱 프로그레스 | Graph 페이지 (기본) | 상세 프로그레스바 + 실시간 통계 |
| 데이터 소스 커넥터 | Collections (문서 단위) | 서비스별 커넥터 카드 (Slack, Notion 등) |
| 벡터DB 상태 패널 | 없음 | 문서 수, 용량, 관리 버튼 |

### 3.3 실제 대시보드가 더 나은 점

| 기능 | 현재 대시보드 | Sand Dune 프로토 |
|------|-------------|-----------------|
| Dark mode | O (light/dark 토글) | X (라이트 모드만) |
| i18n | O (ko/en) | X |
| API 연동 | O (실제 CRUD) | X (정적 HTML) |
| 반응형 | O (Tailwind) | 부분적 |
| 검색 모드 | Hybrid/Vector/FTS/Graph 선택 | 없음 |
| 타입 안전성 | TypeScript | 없음 |

## 4. 흡수 권장 사항

### 즉시 적용 가능 (디자인 컨셉 파일)

1. **컨셉 목업에 멀티뷰 추가** — 기존 1뷰(문서관리) → 최소 3뷰 (문서, 채팅, 모니터)
2. **Lucide 아이콘 적용** — 이모지 → SVG 아이콘으로 품질 향상
3. **풀페이지 프로토타입 포맷** — 인기 테마 3~5개를 Sand Dune처럼 풀 프로토로 확장

### 실제 대시보드 개선 고려

1. **RAG 채팅 출처 인용 UI** — Search 페이지에 채팅형 UI + 출처 인라인 블록 추가
2. **인덱싱 모니터 강화** — Graph 페이지에 프로그레스바 + 실시간 통계 추가
3. **데이터 소스 커넥터 UI** — Collections에 서비스별 아이콘 카드 패턴 도입
4. **벡터DB 상태 대시보드** — Settings 또는 별도 패널로 DB 상태 표시
